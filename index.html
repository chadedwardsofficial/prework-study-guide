HTML

  • HTML stands for "Hyper Text Markup Language"
  • HTMl is considered the foundation for all webpages, it provides structure to the webpage.
  • Every html document starts with a " !DOCTYPE in <>"
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.

  • There are 6 types of headers varying in size. These will be in ascending order ex. H1..H2..H3.. with H1 being the largest and thusfore. These will be wrapped in "<>"
  • Elements such as "Body" and "Head" are commonly referred to as Parent Elements and all elements within these tags are considered their Child Elements.
  • To represent a body of text or a paragraph; you use p in <> brackets.
  • Semantic elements are elements where the name describes its content and role it plays within the webpage. For example: header, footer, main, section.
  • To make an unordered list of items you use the tag "ul" in <> brackets. Inbetween these tags you then use a smaller tag to make your bulleted point list using "li" in <> brackets.
  • The same list can be made using numbers which is considered an "Ordered List"; in which this case we would use "ol" in <> tags.
  • You can also insert images and hyperlinks.
  • Image insert codes goes as follows: img src="insert-link-to-img" in a pair of <>Brackets. Within the "" you will paste your link to code wether it be a .png file or a link to another website's image.
  • To insert a hyper link, we use the "a" anchor tag. a href="url-of-website". href= is a code to refer to the link you are sending the user to. The entire code will also be wrapped in <> brackets.

Git

  • Git is a commonly used CLI (Command Line Interface) software that allows the user to track changes in their computer files by using Version Control
  • Version Control is a great way in which developers can keep track of every iteration of their code so they can track their mistakes in current or past codes.
  • Git uses basic coded language to help navigate through your directory on a text based level. Similarly to how we navigate a folder on our desktop, that is done through a visual way also known as GUI (Graphical User Interface)
  • Using git allows a developer to also interact with GitHub, a database online that helps all collaboraters work together on a project.
  • Git has basic commands such as:
    • ls: list of directories
    • cd "name of file": takes you to named file directory
    • pwd:print working directory
    • mkdir "name of file": makes a new directory in your listed using the name you inserted after mkdir.
  • There are many more commands for Git that also allow you to clone(copy), modify and push repositories to the GitHub Database.
